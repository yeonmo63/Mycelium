# Mycelium Enterprise - Deployment & Security Guide

이 문서는 Mycelium 시스템을 실제 운영 환경에 구축하고 보안을 강화하기 위한 가이드입니다.

## 1. 초기 시스템 설치 및 보안 설정 (Setup Wizard)

시스템을 처음 실행하면 **시스템 설정 마법사**가 나타납니다. 다음 단계를 통해 보안이 강화된 환경을 구성하십시오.

### 1단계: 데이터베이스 연결

- PostgreSQL 서버의 신뢰할 수 있는 정보를 입력하십시오.
- 기본 포트(5432) 대신 사용자 정의 포트를 사용하는 것을 권장합니다.

### 2단계: AI 서비스 설정 (선택사항)

- Gemini API 키를 입력하여 지능형 데이터 분석 기능을 활성화할 수 있습니다.

### 3단계: 시스템 보안 키(JWT Secret) 생성 **[중요]**

- 로그인 세션을 보호하는 핵심 키를 생성합니다.
- 시스템에서 **무작위로 생성된 키**를 그대로 사용하거나, 더 강력한 무작위 문자열을 입력하십시오.
- 이 키는 `.env` 파일에 기록되며, 외부에 절대 유출되지 않도록 하십시오.

## 2. 보안 운영 수칙

### 🔑 비밀번호 관리

- 최초 로그인 후 `admin` 계정의 비밀번호를 즉시 변경하십시오.
- 관리자 비밀번호는 최소 12자 이상, 특수문자를 포함하여 복잡하게 설정하십시오.

### 🛡️ 보안 상태 점검

- 정기적으로 **[설정 및 관리] > [보안 상태 점검]** 메뉴에 접속하여 시스템의 취약점을 확인하십시오.
- JWT 비밀키가 기본값이거나 안전하지 않은 경우 시스템이 즉시 경고를 표시합니다.

### 📝 감사 로그 모니터링

- **[설정 및 관리] > [시스템 감사 로그]**를 통해 중요한 데이터 변경이나 접근 이력을 정기적으로 검토하십시오.
- 의심스러운 IP에서의 접근이나 세션 로그가 상시 기록됩니다.

### 📱 기기 보안 관리

- **[로그인 기기 관리]** 메뉴를 통해 현재 사용 중인 세션을 실시간으로 확인하십시오.
- 분실했거나 미승인된 기기가 접속 중일 경우 즉시 **[원격 로그아웃]**을 수행하십시오.

## 3. 백업 및 유지보수

- **자동 백업**: 시스템은 6시간마다 자동으로 데이터베이스를 백업합니다.
- **외부 저장소**: 랜섬웨어나 하드웨어 고장을 대비해 **[데이터 백업 및 복구]** 메뉴에서 별도의 하드디스크나 네트워크 드라이브 경로를 지정하십시오.
- **최적화**: 데이터가 많이 쌓여 시스템이 느려질 경우 **[데이터베이스 최적화 (VACUUM)]**를 수행하십시오.

## 4. 환경 변수 (.env) 설정 예시

설치 마법사 완료 시 생성되는 `.env` 파일의 주요 항목입니다. 수동으로 수정할 경우 프로그램을 재시작해야 합니다.

```env
DATABASE_URL=postgres://user:pass@host:port/name
JWT_SECRET=당신의_강력한_비밀키_문자열
GEMINI_API_KEY=AI_서비스_키 (선택)
PORT=3000
RUST_LOG=info,sqlx=warn
```
